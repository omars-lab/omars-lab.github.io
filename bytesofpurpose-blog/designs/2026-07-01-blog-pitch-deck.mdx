I built a pitch deck for this blog inside a design system, then brought it home as a real,
navigable deck living inside this post. It is not a screenshot and not an embedded slide
service: it is [reveal.js](https://revealjs.com) driving on-brand slides, styled from the exact
same tokens as the prose you are reading. Use the arrow keys, press <kbd>Esc</kbd> for the slide
grid, or <kbd>F</kbd> for fullscreen.

## Key decisions

The deck above is a `<SlideDeck>` component wrapping seven `<Slide>` children. Nothing about it is
a special one-off: `<SlideDeck>` is a reusable primitive registered globally, so any post can embed
a deck the same way. Three decisions carry the whole thing, and each was a genuine trade-off.

**reveal.js loads in the browser, on this page only.** Docusaurus server-renders every page, and
reveal.js touches `window` the moment it is imported, so a top-level import would break the build.
The component wraps everything in `<BrowserOnly>` and `import()`s reveal.js, its notes plugin, and
its base CSS lazily inside a `useEffect`. The rest of the site never downloads a byte of it.

**No reveal.js theme ships: our tokens are the theme.** reveal.js comes with a dozen themes; the
deck loads none of them. Instead the component's CSS module restyles the reveal surface from the
site's own design-system tokens: Fraunces for display type, Geist for body, the tea pastels as
accent fills, deep green for the full-bleed cover and CTA slides. So the deck is the same brand as
the prose around it, and it flips correctly in dark mode for free, because the tokens do.

**It stays a deck, not a slideshow gimmick.** reveal.js gives real presentation affordances:
keyboard navigation, an overview grid on <kbd>Esc</kbd>, a speaker-notes view on <kbd>S</kbd> (every
slide above carries `notes`), and PDF export by appending `?print-pdf` to the URL. Reduced-motion
visitors get instant slide changes instead of the slide transition.

If you want to build one, the `author-slide-deck` skill walks through the component API, the
token-mapping rules, and the MDX gotchas, using this very post as the worked example.
