The vocabulary I use to organize CLI tooling around **roles, skills, and
activities**, plus the **workflows** that span them. These nest into each other,
so this page looks at them through three lenses:

1. **[Glossary](#1-glossary)** — each term defined once.
2. **[The model](#2-the-model)** — how they compose, plus the path convention that lays them out.
3. **[Distinctions](#3-distinctions-x-vs-y)** — the contrasts that are easiest to confuse.

## 1. Glossary

| Term | Definition | The question it answers |
|---|---|---|
| **Role** | A set of skills and activities that together fulfil a function. | *Who am I being?* |
| **Skill** | A capability composed of activities and possibly simpler skills. | *What can I do?* |
| **Activity** | The smallest unit of work — atomic, not decomposed further. | *What's the actual step?* |
| **Workflow** | Multiple roles working in unison toward a larger outcome. | *How do roles combine?* |

## 2. The model

Roles, skills, and activities nest by *granularity* — a role at the top, atomic
activities at the bottom, with skills (which can contain sub-skills) in between.
Workflows sit above roles, coordinating several at once.

- A **role** involves a set of skills and activities.
- A **skill** is composed of many different activities and possibly simpler skills.
  - **Complex skills** break down into simpler skills and activities.
  - **Simple skills** are just a set of activities.
- A **workflow** involves multiple roles working in unison.

Each role and skill has an *overview* plus the things it contains:

```
Role  →  /role/overview
           skillset
           activity-set

Skill →  /role/skill/overview
           sub-skills
           activities
           parentSkill
```

### Path convention

Roles, skills, and activities map onto a directory-like path. Complex activities
can nest, and each level has its own overview:

```
/role/hat
/role/hat/overview
/role/hat/complex-activity/overview
/role/hat/complex-activity/simple-activity
/role/hat/complex-activity/complex-activity
/role/hat/complex-activity/complex-activity/overview
/role/hat/simple-activity
```

When organizing files, categorize each one individually first, then group them
at the end — rather than recursively expanding everything up front.

## 3. Distinctions (X vs. Y)

### Role vs. Skill — *identity vs. capability*
- A **role** is who you're being; it bundles many skills and activities.
- A **skill** is one capability that role draws on. One role → many skills.

### Skill vs. Activity — *capability vs. step*
- A **skill** can be decomposed (into sub-skills and activities).
- An **activity** is atomic — the smallest step, not broken down further.

### Complex skill vs. Simple skill — *nested vs. flat*
- A **complex skill** contains sub-skills as well as activities.
- A **simple skill** is just a flat set of activities.

### Role vs. Workflow — *one actor vs. many*
- A **role** is a single actor's bundle of capability.
- A **workflow** coordinates **multiple roles** working in unison.
