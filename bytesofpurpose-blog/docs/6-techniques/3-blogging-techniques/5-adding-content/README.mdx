## Understanding Content Types

Each content type has specific characteristics and use cases:

### Blog Posts
**Purpose**: Discovery, storytelling, capturing moments in time

**When to use**: 
- Personal experiences and lessons learned
- Time-sensitive content
- Opinion or analysis pieces
- Story-driven content

**Location**: `blog/` directory

### Changelog Entries
**Purpose**: Track changes, features, and improvements systematically

**When to use**:
- New features or components
- Bug fixes
- Infrastructure improvements
- Content additions
- Refactoring work

**Location**: `changelog/content/` or `changelog/development/` directories

### Documentation
**Purpose**: Reference material and problem-solving guides

**When to use**:
- Step-by-step guides
- API documentation
- Configuration instructions
- Durable knowledge that won't become outdated quickly

**Location**: `docs/` directory

### Design Posts
**Purpose**: Document design decisions and architectural insights

**When to use**:
- Design decisions and rationale
- Architecture documentation
- Visual design documentation
- System design explanations

**Location**: `designs/` directory
