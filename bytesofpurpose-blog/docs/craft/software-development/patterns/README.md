# Patterns

This is the durable index of the **architectural patterns** I reach for: how the tools and
pieces of a system **compose** into a working whole. Each entry answers the same question in
a different setting, "how do these fit together?", from a build pipeline to a documentation
stack.

Unlike a project log, a pattern is not a dated thing I did; it is a return-to reference that
stays true across projects. So it lives here in Craft, not on the Initiatives blog. This page
groups the patterns by area so you can scan the whole set at once. Move a pattern between areas
by editing its `area` frontmatter, and it re-sorts here automatically.
