# 📋 Product Management

**What to build, and why.** This topic is the idea→ship lifecycle — the place where
work is shaped before (and while) it gets built. The things actually built live over in
[Software Development](/docs/development); ideas here link to their executions there.

## The lifecycle

Work moves through stages, roughly in this order:

1. **💡 Ideas** — initial thoughts and wishlists. *What do I want to make?*
2. **🔬 Research** — deep dives and investigation before committing.
3. **🧪 Proof of Concepts** — small builds that validate an approach.
4. **⚗️ Experiments** — hypothesis-driven, measured tests (A/B, RCTs).
5. **🚀 Initiatives** — researched and designed, ready to start (not yet active).
6. **🔨 Projects** — initiatives now in active development.
7. **🗺️ Roadmaps** — how I'm improving what already exists.

## Idea ↔ execution mapping

An idea here links to what got built under an **`## Execution`** section; the built
artifact in [Software Development](/docs/development) links back under an **`## Idea`**
section. That way you can trace any idea to its execution and vice-versa.
