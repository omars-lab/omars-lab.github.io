Evaluating Docs vs Blogs Blog Post

January 15, 2025 · 2 min read
Omar Eid
Principal AI Architect & Entrepreneur

Blog Post: 2025-01-31-docs-vs-blogs.md
Focus: Content strategy and format selection

Core Questions​

✅ Problem Clarity​

  • Clear distinction: Does the post clearly explain the difference between docs and blog posts?
  • Use case examples: Does it provide specific examples of when to use each format?
  • Decision framework: Does it offer a practical framework for choosing between them?

✅ Practical Value​

  • Actionable guidance: Can readers immediately apply the decision criteria?
  • Real examples: Does it include concrete examples of good docs vs good blog posts?
  • Framework utility: Is the decision framework easy to use and remember?

✅ Content Quality​

  • Clear writing: Is the content easy to understand and follow?
  • Logical structure: Does it flow logically from problem to solution?
  • Engaging tone: Is it written in an engaging, helpful style?

Specific Criteria​

Content Strategy Focus​

  • Format comparison: Clearly compares docs vs blog posts
  • Decision criteria: Provides clear criteria for choosing between formats
  • Examples: Includes specific examples of each format
  • Framework: Offers practical decision-making framework

Value Delivery​

  • Immediate applicability: Readers can use the guidance right away
  • Clear benefits: Shows why format choice matters
  • Practical tips: Provides actionable advice for content creators

Writing Quality​

  • Scannable format: Uses headers, bullets, and clear structure
  • Concise explanations: Avoids unnecessary complexity
  • Engaging examples: Uses relatable scenarios and examples

Quality Threshold​

Ready to Publish: 8+ criteria met (80%+)
Needs Revision: 6-7 criteria met (60-79%)
Major Revision: Below 6 criteria met ("under 60%")

Key Questions to Ask​

  1. Would a content creator find this immediately useful?
  2. Does it solve a real problem content creators face?
  3. Is the decision framework practical and memorable?
  4. Are the examples clear and relatable?

Evaluating My Approach Blog Post

January 15, 2025 · 5 min read
Omar Eid
Principal AI Architect & Entrepreneur

Blog Post: 2025-09-27-my-approach.md
Focus: Personal problem-solving methodology and approach demonstration

This rubric evaluates whether the "My Approach" blog post effectively demonstrates your core problem-solving methodology and approach to handling complex situations.

Core Questions​

✅ Problem-Solving Demonstration​

  • Clear methodology: Does it show a systematic approach to problem-solving?
  • Specific examples: Does it include concrete examples of applying the approach?
  • Process clarity: Is the step-by-step process easy to follow?

✅ Ambiguity Navigation​

  • Unclear situations: Does it show how you handle ambiguous or unclear problems?
  • Decision framework: Does it demonstrate your decision-making process?
  • Information gathering: Shows how you collect and evaluate information?

✅ Impact Creation​

  • Value delivery: Does it show how your approach creates meaningful impact?
  • Measurable results: Includes specific outcomes or improvements?
  • Stakeholder benefit: Shows how others benefit from your approach?

Specific Criteria​

Methodology Clarity​

  • Systematic process: Shows a clear, repeatable approach
  • Step-by-step: Breaks down the process into actionable steps
  • Decision points: Identifies key decision points in the process
  • Adaptation: Shows how you adapt the approach to different situations

Problem-Solving Examples​

  • Real scenarios: Uses actual examples from your experience
  • Context setting: Provides enough background to understand the situation
  • Process application: Shows how you applied your methodology
  • Outcome demonstration: Shows the results of your approach

Personal Insight​

  • Unique perspective: Offers insights that others can't easily replicate
  • Learning reflection: Shows what you've learned from your approach
  • Continuous improvement: Demonstrates how you refine your methodology
  • Teaching value: Provides value for others facing similar problems

Quality Threshold​

Ready to Publish: 8+ criteria met (80%+)
Needs Revision: 6-7 criteria met (60-79%)
Major Revision: Below 6 criteria met ("under 60%")

Key Questions to Ask​

  1. Does this show a clear, systematic approach to problem-solving?
  2. Would others find this methodology useful and applicable?
  3. Does it demonstrate real problem-solving skills with examples?
  4. Is the approach unique and valuable compared to generic advice?

Final Validation Test​

Before publishing, ask yourself these final questions:

Core Problem-Solving Test:​

  • Does this post clearly show how I navigate ambiguity?
  • Does it demonstrate my technical decision-making process?
  • Does it show the impact I created?
  • Does it reveal how I work with others?
  • Does it show my learning and adaptation process?

Value Test:​

  • Would I want to read this post if someone else wrote it?
  • Does this post make me look like someone I'd want to work with?
  • Would this post help someone solve a real problem?
  • Does this post demonstrate expertise without being arrogant?

Impact Test:​

  • Does this post position me as a problem-solver?
  • Does it show I can handle complex, ambiguous situations?
  • Does it demonstrate I create meaningful impact?
  • Would this post influence someone's decision to work with me?

Content Planning Template​

When planning the "My Approach" blog post, use this template to ensure you're covering your core themes:

Problem Statement: What ambiguous situation did I face?

Technical Decisions: What tradeoffs did I evaluate and how did I decide?

Impact Created: What measurable or observable results did I achieve?

Collaboration: Who did I work with and how did we solve this together?

Learning: What did I learn and how would I approach this differently?

Key Insight: What's the main takeaway that others can apply?

Combined Evaluation Process​

For Complete Evaluation​

  1. Complete this specific rubric for problem-solving methodology evaluation
  2. Use general-blog-evaluation.md for content quality and professional positioning (45 points)
  3. Add scores together for total evaluation
  4. Apply quality thresholds based on combined score
  5. Combine feedback from both evaluations for comprehensive guidance

Evaluating My Contributions Blog Post

January 15, 2025 · 5 min read
Omar Eid
Principal AI Architect & Entrepreneur

Core Questions​

Timeline Structure and Flow​

  • Chronological Organization: Does the timeline start with most recent contributions and flow to older ones?
  • STAR Summary Format: Are timeline items concise STAR (Situation, Task, Action, Result) summaries?
  • Link to Details: Does each timeline item link to a detailed section later in the document?
  • Visual Hierarchy: Is the timeline visually clear with proper color coding and alignment?

Contribution Impact and Clarity​

  • Specific Impact Metrics: Are concrete, measurable results included (e.g., "40% improvement", "reduced costs by 60%")?
  • Problem-Solving Demonstration: Does each contribution show how ambiguity was navigated and decisions were made?
  • Stakeholder Impact: Is it clear who benefited from each contribution (users, team, business, customers)?
  • Career Progression: Does the timeline show clear growth and increasing responsibility over time?

Professional Storytelling​

  • Key Verbs Usage: Are action verbs used effectively (invented, aligned, developed, designed, led, optimized, etc.)?
  • Contribution Buckets: Are contributions organized into clear categories (technical, leadership, process, impact)?
  • Stakeholder Icons: Are visual indicators used to show different parties worked with (engineers, PMs, customers, etc.)?
  • Narrative Coherence: Does the overall story flow logically and build a compelling professional narrative?

Specific Criteria​

Timeline Content Quality​

  • Concise Summaries: Each timeline item is 2-3 sentences maximum
  • STAR Structure: Situation, Task, Action, Result clearly defined
  • Impact Focus: Results and outcomes are prominently featured
  • Link Integration: Clear links to detailed sections for deeper dive

Detailed Sections Quality​

  • Comprehensive Coverage: Detailed sections expand on timeline summaries
  • Technical Depth: Sufficient detail for technical audience
  • Decision Context: Background and reasoning for key decisions
  • Learning Integration: Lessons learned and knowledge transfer

Visual and Structural Elements​

  • Color Coding: Different companies/roles have distinct colors
  • Icon Usage: Stakeholder icons clearly show collaboration partners
  • Contribution Categories: Clear buckets for different types of contributions
  • Professional Layout: Clean, readable, and visually appealing

AI Metadata Integration​

  • Timeline Structure Guidance: AI metadata explains how to structure timeline
  • Grading Criteria Reference: Clear reference to separate grading rubric file
  • Update Instructions: Instructions for maintaining timeline accuracy
  • Content Guidelines: Guidelines for adding new contributions

Key Questions​

  1. Does the timeline effectively showcase professional growth and impact?
  2. Are the STAR summaries concise yet compelling?
  3. Do the detailed sections provide sufficient depth for interested readers?
  4. Is the visual design clear and professional?
  5. Does the content demonstrate consistent problem-solving approach?

Combined Process​

This rubric should be used in combination with the general blog evaluation criteria to ensure both content quality and professional impact are properly assessed.

Quality Threshold​

  • Ready to Publish: 12+ criteria met (80%+)
  • Needs Minor Revision: 9-11 criteria met (60-79%)
  • Needs Major Revision: Below 9 criteria met ("under 60%")

Content Planning Template​

Timeline Structure​

  • Start with most recent contribution
  • Use concise STAR summaries (2-3 sentences)
  • Include clear links to detailed sections
  • Use color coding for different companies/roles
  • Add stakeholder icons for collaboration partners

Contribution Buckets​

  • Technical: Architecture, performance, system design
  • Leadership: Team management, process improvement, mentoring
  • Innovation: New features, AI integration, automation
  • Impact: User experience, business value, cost optimization

Key Verbs to Include​

  • Technical: Designed, developed, optimized, migrated, implemented
  • Leadership: Led, aligned, mentored, facilitated, coordinated
  • Innovation: Invented, created, pioneered, introduced, transformed
  • Impact: Improved, reduced, increased, delivered, achieved

Stakeholder Icons​

  • 👥 Engineering Teams: Technical collaboration
  • 📊 Product Managers: Feature planning and requirements
  • 🎨 Designers: User experience and interface design
  • 👔 Business Stakeholders: Strategic alignment and value delivery
  • 👤 Customers/Users: End-user impact and satisfaction
  • 🏢 Leadership: Executive reporting and strategic decisions

Focus: Company culture analysis and insights

Core Questions​

✅ Cultural Analysis​

  • Value breakdown: Does it clearly explain each of Zapier's 5 core values?
  • Practical examples: Shows how values translate to actual behaviors?
  • Cultural insights: Provides meaningful analysis of the culture?

✅ Comparative Analysis​

  • Company comparison: Does it compare Zapier to other tech companies?
  • Size context: Shows how company size affects cultural values?
  • Unique positioning: Identifies what makes Zapier's approach distinctive?

✅ Practical Value​

  • Applicable insights: Can readers apply these insights to their own work?
  • Decision framework: Helps readers understand different cultural models?
  • Actionable takeaways: Provides concrete guidance for cultural decisions?

Specific Criteria​

Cultural Understanding​

  • Value explanation: Clearly explains each of the 5 core values
  • Behavioral examples: Shows how values translate to specific behaviors
  • Cultural context: Provides background on why these values matter
  • Evolution story: Shows how values have developed over time

Comparative Analysis​

  • Company comparison: Compares Zapier to Amazon, Meta, or other companies
  • Size analysis: Shows how company size influences cultural development
  • Unique insights: Identifies what makes Zapier's approach distinctive
  • When to use: Provides guidance on when Zapier's model applies

Practical Application​

  • Remote work insights: Shows how values support remote-first culture
  • Team building: Provides guidance for building similar cultures
  • Decision making: Helps readers choose between cultural models
  • Implementation: Offers practical steps for applying insights

Quality Threshold​

Ready to Publish: 8+ criteria met (80%+)
Needs Revision: 6-7 criteria met (60-79%)
Major Revision: Below 6 criteria met ("under 60%")

Key Questions to Ask​

  1. Does this provide deep insights into Zapier's culture?
  2. Would someone considering working at Zapier find this valuable?
  3. Does it help readers understand different cultural approaches?
  4. Are the insights actionable and applicable beyond Zapier?
